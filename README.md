## REQUISITOS DEL README.md

Debe incluirse un archivo README.md en la raíz del repositorio Git. Su propósito es
permitir que cualquier persona que no esté familiarizada con el proyecto (pares, personal, responsables de selección, etc.) pueda entender rápidamente de qué trata el proyecto, cómo ejecutarlo y dónde encontrar más información sobre el tema.

El README.md debe incluir, como mínimo:
• La primera línea debe estar en cursiva y decir: Este proyecto ha sido creado como parte del currículo de 42 por <login1>[, <login2>[, <login3>[...]]].
• Una sección de "Descripción"que presente claramente el proyecto, incluyendo su
objetivo y una breve visión general.
• Una sección de "Instrucciones"que contenga cualquier información relevante sobre
compilación, instalación y/o ejecución.
• Una sección de "Recursos"que enumere referencias clásicas relacionadas con el
tema (documentación, artículos, tutoriales, etc.), así como una descripción del uso
de IA, especificando para qué tareas y en qué partes del proyecto se ha utilizado.

- Podrían requerirse secciones adicionales dependiendo del proyecto (por
ejemplo, ejemplos de uso, lista de características, decisiones técnicas, etc.).Cualquier contenido extra requerida se listará explícitamente a continuación.

• También debe incluirse una descripción detallada de la librería creada para este
proyecto.
